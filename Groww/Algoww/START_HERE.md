## Instructions for Copilot

You are resuming an ongoing DSA mentoring journey. Before responding, read the following files from this workspace:

1. **`MENTOR_INSTRUCTIONS.md`** — Your persona, session flow, behavior rules. Follow these STRICTLY.
2. **`PROGRESS_TRACKER.md`** — The student's current week, day, confidence levels, and daily log. This tells you WHERE they left off.
3. **`REVISION_LOG.md`** — Quiz history, spaced repetition queue, and concept quick-reference. Check if anything is due for revision.
4. **`2_MONTH_PLAN.md`** — The overall 8-week schedule. Use this to know what topic/problems come next.
5. **The current topic's `README.md`** — e.g., `01_Arrays_Hashing/README.md`. This has the pattern explanation, templates, and problem list for the active topic.

## What to do

1. Check `PROGRESS_TRACKER.md` to find the **current week, day, and last completed session**
2. Check if any **revision or quiz is due** from `REVISION_LOG.md`
3. Pick up exactly where we left off — either:
   - Continue with the next problem in the current topic
   - Start Phase 0 (README walkthrough) if it's a new topic
   - Run a revision/quiz session if one is due
4. Follow the **Session Flow** from `MENTOR_INSTRUCTIONS.md` (Phase 0 → 1 → 2 → 3 → 4 → 5)
5. After the session, update `PROGRESS_TRACKER.md` and `REVISION_LOG.md`
